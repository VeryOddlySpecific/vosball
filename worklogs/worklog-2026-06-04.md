## Files Changed
**New `webapp/ui.py` — shared LCARS components (pure rendering, no data/engine):**
- `metric_row(specs)` — replaces the repeated `m = st.columns(n); m[i].metric(...)`
  idiom with a `[(label, value), …]` list.
- `lcars_section(title)` — themed section-header bar (cap + uppercase label +
  trailing rule); drop-in for bare `st.markdown("**…**")`.
- `chip_line(items)` — the ` · `-joined caption (bio/tier), blanks dropped.
- `identity_header(name, cells)` — large name + a horizontal LCARS data bar; each
  `(label, value)` cell is a connected pill, last grows to fill. HTML-escaped.
- `lcars_chips(items, variant)` — bio/tier rendered as styled pill chips
  (`variant="tier"` for a distinct accent).

**`webapp/app.py` — Player Card restructure + theming:**
- Added `.lcars-section`, `.lcars-id` (name + data bar), and `.lcars-chips` CSS to
  `build_theme_css` (both palettes). New classes are emitted *only* by the new
  components, so no bleed to other pages.
- Refactored `_render_card` + `_render_contract` to use the components: replaced
  ~6 hand-rolled metric blocks and ~9 bare bold headers. Headers with a trailing
  descriptor became a section bar + caption.
- **Identity panel:** name + bio/tier chips + headline VOS now bound in one
  bordered `st.container`. The identity bar's 3 cells are **Bat / Thr**
  (`raw["Bats"]/["Throws"]`), **Pos** (`row["Pos"]`), and **Qualified**
  (`Projected_Viable_Pos_List` minus current pos; blank for pitchers).
- **Component scores (hitters):** now 3 columns — Batting | Defense | Baserunning
  — each headed by its score with that component's scouted-ratings table beneath.
  Batting/Defense/Baserunning tables moved up out of the "Scouted ratings"
  expander (which now holds just Position ratings + Personality).
- **Defense table:** new `_paired_ratings_df` helper renders a 4-column
  infield | outfield table (`HITTER_DEFENSE_FIELDS[:4]` vs `[4:]`); catcher skills
  (CBlk/CArm/CFrm) ride the outfield column (blank/hidden for non-catchers).
- **Whole-number display:** added `_whole()` (rounds numerics, passes blanks/text
  through). Rounded headline VOS, component scores, pitcher Ability/Arsenal, the
  role-comparison score rows (`_ROLE_SCORE_METRICS`/`_ROLE_WAR_METRICS` VOS Ceiling
  → prec 0), all skill-rating tables, and the positional-scores table. Left
  Adjustments and WAR with decimals.
- `_ratings_into` now skips the caption when `label` is empty (lets the 3-column
  tables sit captionless under their metric heading).

## Deviations from Plan
- **Phasing collapsed slightly.** The plan was Phase 1 = pure component extraction
  (no visual change), Phase 2 = identity header. In practice Phase 1's section
  headers became LCARS bars immediately (a small intentional visual delta), and
  Phase 2+ grew several user-driven tweaks (identity data bar, 3-col components,
  4-col defense, whole-number rounding) rather than a single restructure.
- **Hit the imported-module hot-reload trap twice.** Editing/adding `ui.py` did
  not hot-reload (Streamlit only re-runs the main script); the running server kept
  a stale `ui.py` and rendered the cells list as plaintext. Fix each time: kill the
  stale `python.exe`/Streamlit processes and relaunch. Rule established: editing
  `app.py` hot-reloads; touching any imported module needs a full restart.
- **Catcher fields** in the Defense split are neither infield nor outfield —
  decided to ride them along the outfield column (hidden for non-catchers) rather
  than drop them; flagged for the user, left as-is.
- Phase 3 (propagating the components to the other pages) was scoped but **not
  started** — this session stayed on the Player Card per the "one page first"
  choice.
