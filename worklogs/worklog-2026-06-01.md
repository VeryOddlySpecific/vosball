# Worklog — 2026-06-01

## Summary
Reworked the Streamlit web UI so evaluations are siloed per league (fixing
cross-league data bleed), the active league flows from the hub into every module
with auto-run on first visit, cold boot lands on a new Home league-select screen,
and the League Hub gained a "Pull fresh ratings" button with live progress. All
delivered in test-backed increments; added a new AppTest-based webapp suite.

## Files Changed
- `webapp/state.py` (new) — shared per-league evaluation silo + active-league
  helpers (`results_silo`, `set_result`, `get_result`, `clear_results`,
  `active_league`, `active_result`) and `drop_result` (evict one league after a
  data refresh). Imports only streamlit → no import cycle.
- `webapp/scoring.py` (new) — extracted data-discovery + scoring from app.py
  (`discover_leagues`, `default_scale_for`, `park_factors_path_for`,
  `player_data_mtime`, `evaluate`, `to_csv_bytes`, module-level cached
  `cached_eval`) plus `autorun_result` (offline-safe first-visit scoring: smart
  scale, park-if-shipped, no draft/contracts/WAR). One shared cache for all pages.
- `webapp/home.py` (new) — cold-boot Home/league-select landing: expanded,
  status-colored league tiles; picking one sets the active league and opens its
  hub. Reuses status.py's cached export check + league.py settings.
- `webapp/fetch.py` (new) — generator `fetch_league_ratings(league)` that drives
  the StatsPlus /ratings two-step async flow (reusing fetch_player_data.py's
  building blocks) and yields progress/done/error events between polls.
- `webapp/app.py` — removed the single global `result` slot; reads/writes the
  silo; Eval Browser league picker now bound to `selected_league` (hub/chip flow
  in); auto-run on first visit; Player Card resolves via active league; Home
  registered as default page + cold-boot gate; per-league rating-scale/park
  re-seed on league switch; defensive "no PlayerData" warning; removed unused
  `os`/`tempfile`/`evaluate` imports; helpers now imported from state/scoring.
- `webapp/depth.py`, `webapp/prospects.py` — resolve via `active_result()` and
  auto-run the active league on first visit.
- `webapp/league.py` — League Hub now renders ratings freshness + "⟳ Pull fresh
  ratings" button driving an `st.status` progress panel; evicts the league's
  cached eval on success.
- `webapp/tests/test_webapp.py` (new) — standalone AppTest suite (no pytest;
  mirrors tests/test_golden.py); export-status preflight stubbed for offline runs.

## Deviations from Plan
- Original Phase 4 left the eval page auto-running the first league on cold boot.
  Per user request, replaced that with a dedicated Home league-select screen
  (new `home.py` + default page + cold-boot redirect gate) so nothing is scored
  until a league is chosen.
- Pulled the data/scoring helpers out of app.py into `scoring.py` (not in the
  original plan) to avoid a `__main__`-vs-module double-import when page modules
  needed `autorun_result`, and to keep one shared scoring cache.
- Fresh-ratings pull (not in the original 5-phase plan) added as a follow-on
  feature at the user's request; implemented as a progress-yielding generator so
  the UI can stream status during the multi-minute export.
