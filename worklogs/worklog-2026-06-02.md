# Worklog — 2026-06-02

## Summary
Built out the free-agent workflow: depth charts now record which eval they were
built from, the FA scan auto-regenerates stale depth batches, and a new
in-process **Free Agents** web-UI page surfaces biggest-holes-first targets with
best-fit FAs — including a pro-service-time gate that keeps amateur/draft-eligible
players out of the recommendations.

## Files Changed
- `depth_chart.py` — **Phase A (eval provenance).** Added `eval_ts_from_path()`
  and `write_depth_meta()`; threaded `eval_path` into `run_one_level`; stamped
  `source_eval`/`source_eval_ts` into the `starter_gaps` sidecar; `main()` now
  writes a per-batch `{org}_{ts}_depth_meta.json` (single- and multi-level alike)
  into `{league}/depth/` — per-league siloed, archives with the batch.
- `free_agent_market.py` —
  - **Phase B (freshness guard).** Added `read_depth_meta`,
    `depth_batch_source_eval_ts`, `regenerate_depth_batch` (subprocess, mirrors
    `run_depth_chart_all`; forces `--org-code` to the FA slug, replays recorded
    min-comp), `ensure_fresh_depth_batch`; wired into `run_multilevel`; new
    `--no-auto-refresh` flag; single-level mode left fresh-by-construction with a
    clarifying log. Added `subprocess`/`sys` imports.
  - **Phase C (fit core).** `score_fa_records` (in-process FA scoring,
    ratings-only default) and `compute_fa_fit` (biggest-holes-first: weak slots
    ranked by gap vs threshold, top-N best-fit FAs per hole) — pure data, no IO,
    shared by UI + CLI.
  - **Service-time gate.** `pro_service_days_of`; extended `score_fa_records`
    with `players_lookup` / `min_pro_service_days` / `exclude_retired` (drops
    retired; conservative gate: unknown service ⇒ excluded).
- `webapp/free_agents.py` (**new**) — Free Agents page. In-process, ratings-only,
  mirrors `depth.py` (org/level helpers reused). Sidebar: org, level, weak-spot
  threshold (defaults to league `min_comp`), FAs-per-need, **Min pro service
  (days)** (default 1). Loads cached `/players` for the service gate; warns when
  unavailable. Renders Priority Needs + per-need candidates + full pool.
- `webapp/app.py` — registered the Free Agents page (import, `st.Page`, `_pages`,
  nav list).
- `webapp/league.py` — flipped the League Hub "Free Agents" tile from placeholder

# Worklog — 2026-06-02 (session 2: Trade Targets)

## Summary
Shipped the **Trade Targets** module end-to-end: a new in-app page that pulls the
league-wide `/tradeblock` and ranks it as a shopping list against your org's
needs (biggest holes first), mirroring how Free Agents was built. The heavy
analytical logic already existed on the CLI (`trade_targets.py`); this session
added token-based auth for `/tradeblock`, extracted a pure scoring core shared by
CLI + UI, built the web page, wired it into the app + hub, and validated live.

## Files Changed
- `trade_targets.py` —
  - **Phase 0 (token auth).** `fetch_tradeblock` now takes a `token` param and
    appends `?token=` (token wins over cookie; cookie/none remain fallbacks).
    The token is kept out of the cache key and log lines (token-less `url` for
    caching/logging, separate `req_url` for the request) so it never lands in a
    filename or on screen. New `resolve_token(league)` reuses
    `fetch_player_data.load_token_for` (local import, degrades to None). CLI
    `main()` now auto-resolves the token when no `--cookie` is given — an
    explicit `--cookie` still wins.
  - **Phase 1 (pure core).** New `build_trade_targets(args, cfg, eval_rows,
    tradeblock_pids, levels, year, hitter_stats, pitcher_stats)` →
    `{targets, scored, all_org_hitters, all_org_pitchers}`. Lifts the needs →
    candidate-eval → match → score → categorize → filter pipeline out of
    `main()`; no I/O. `main()` refactored to call it (behavior preserved: same
    rc=2 empty-roster guard, same logging/outputs).
- `webapp/trade_targets_page.py` (**new**) — the Trade Targets page. Named
  `*_page` deliberately to avoid shadowing the root `trade_targets.py` core on
  `sys.path` (APP_DIR precedes ROOT). Cached `fetch_block_pids` (token-authed,
  per-day disk cache + session cache, cleared by a refresh button) and
  `build_targets_context` (deep-copies silo rows before the `/players` override
  so the Eval Browser isn't corrupted; applies override + retired filter,
  fetches stats over all lids — full CLI fidelity — then runs the core with
  floors wide open). Page-level `_apply_filters` mirrors the core filter so the
  min-composite / no-need controls stay instant. Renders summary metrics, a
  collapsible **Your needs** table, the **🎯 Shopping list — best fits first**
  (fit-sorted), and per-category expanders. Graceful edges: no base URL, empty
  block with vs without a token (distinct guidance), no org pool, stats-missing.
- `webapp/app.py` — imported `trade_targets_page`, registered its `st.Page`
  (`url_path="trade_targets"`), added it to `_pages` and the nav list. Added a
  comment explaining the page-vs-core naming-collision rule.
- `webapp/league.py` — flipped the League Hub "Trade Targets" tile from
  placeholder (`page: None`) to the live `"trade_targets"` page.
- `tests/test_trade_targets_core.py` (**new**) — offline tests (22): Phase-0
  auth wiring (token URL construction, token-wins-over-cookie, cookie-only path,
  `resolve_token` safety; `urlopen` monkeypatched) + Phase-1 core against the
  `ndl` eval with a stubbed pid list (needs shape, valid tiers/categories,
  ML/AAA scoping, own-org exclusion, filtering narrows the pool, empty-block
  no-crash).
- `webapp/tests/test_webapp.py` — added `trade_targets` to the expected-pages set
  and a new offline smoke test (`test_trade_targets_page_renders_shopping_list`)
  that stubs the two network seams and asserts the fit-sorted list renders.

## Deviations from Plan
- The original premise of the planning question (a session **cookie** is needed
  for `/tradeblock`) was wrong — the user pointed out the configured API token
  works as a `?token=` parameter. That collapsed the whole "how does the page get
  the block" fork: no cookie UI, the page is as seamless as Free Agents. Drove
  Phase 0's token support.
- User chose **full stats fidelity** for the page (not the ratings-only v1 that
  Free Agents shipped with), so `build_targets_context` fetches stats over all

- **Naming collision** caught during Phase 3: a page named `trade_targets.py`

  own `import trade_targets as tt` would import itself. Renamed the page to
  `webapp/trade_targets_page.py` — matches the existing depth/free_agents ≠
  depth_chart/free_agent_market convention.

# Worklog — 2026-06-02 (session 3: Farm Value)

## Summary
Shipped the **Farm Value** module: a new in-app page that values every org's farm
system and **ranks them**, so your team reads as "🏆 Ranked 4th of 30" with its
top farm assets beneath. League- and team-siloed. Reuses the existing
`farm_value.py` analysis (prospect_score × VPC, top-12 + 25% tail) plus the
Prospects page's in-process board; added a pure ranked core shared by CLI + UI.

## Files Changed
- `farm_value.py` —
  - **Phase 1 (core).** New `build_farm_values(rank_rows, eval_rows, …)` →
    `{vpc_base, mlb_count, vpc_ok, player_rows, org_rows}`. Pure, no I/O. VPC
    calibration wrapped so a **no-contracts eval falls back to `vpc_base=1.0`,
    `vpc_ok=False`** (model points) instead of raising — ranking stays correct
    because VPC is a global scalar. New `rank_org_values(org_rows)` stamps
    `rank` + `num_orgs` ("4 of 30"), ordered by `farm_value_total`.
  - `main()` refactored to call both; CLI org CSV/MD output gains `rank` and
    `num_orgs` columns (additive). VPC-fallback now logs a clear warning instead
    of crashing the run (em-dash swapped for ASCII to keep Windows console logs
    clean).
- `webapp/farm_value_page.py` (**new**) — the Farm Value page. Named `*_page` to
  avoid shadowing the root `farm_value.py` core on `sys.path` (same lesson as
  Trade Targets). Cached `build_farm_context`: builds the prospect board
  in-process via `pr.compute_prospect_rows` (offline, no service-time), calibrates
  VPC from the **latest on-disk eval** (`dc.find_latest_eval`, `VOS_Career`
  anchor — always present, unlike the legacy `VOS_Potential` alias), runs
  `fv.build_farm_values`. Cache key folds in the VPC eval's name + mtime. Renders:
  **League rank — Nth of M** headline metrics, the full ranked league table (your
  row marked ◀), and your team's top farm assets. `$` when the eval has contracts,
  else a clearly-labeled unitless **farm index** with a hint to run a contracts
  eval. Org selector defaults to your team.
- `webapp/app.py` — imported `farm_value_page`, registered its `st.Page`
  (`url_path="farm_value"`, 💲), added to `_pages` and the nav list.
- `webapp/league.py` — flipped the League Hub "Farm Value" tile from placeholder
  to the live `"farm_value"` page.
- `tests/test_farm_value_core.py` (**new**) — offline core tests (16):
  contracts eval → real VPC + `$` + ranking; no-contracts eval → graceful
  fallback with *identical* ranking; `rank_org_values` ordering + "of N".
- `webapp/tests/test_webapp.py` — added `farm_value` to the expected-pages set
  and a real-page render test (`test_farm_value_page_renders_rankings`, offline
  against ndl).

## Deviations from Plan
- **Dollar values** (user's choice): VPC is calibrated from the **latest on-disk
  eval**, not the silo rows or a network fetch. Key realization that shaped the
  design: VPC is a global scalar, so the **org ranking never needs contracts** —
  only the dollar magnitudes do. So the page always ranks (offline) and shows
  `$` only when the on-disk eval happens to carry contract columns, else a
  unitless farm index. For the current `ndl` eval (generated contracts-off in
  sessions 1–2) this means the index path — verified live.
- The CLI `main()` couldn't be smoke-run end-to-end: it reads a
  `prospect_rankings_*.csv` board and none exist on disk for any league. That
  path is moot for the page (which builds the board in-process), so the core was
  validated via unit tests + a real-data in-process run instead.
