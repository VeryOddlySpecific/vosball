vContracts data flow — extension planner

data source existing tool (reused) vContracts (implemented) vContracts (stub) output

Inputs

players_contract.csv csv
Existing team obligations. Drives the starting cap-calendar ledger.
players.csv / roster csv
Roster identity, age, position, service time. Used to identify extension candidates and apply age-driven shape locks.
evaluation_summary_{league}_*.csv csv
VOS current + potential per player. Produced upstream by vos_v2.py.
depth chart outputs csv/md
Per-position internal replacement ETAs across the planning horizon. Produced by depth_chart.py.
rules_profile.py done
sdmb() / sdmb_post_option_rule(). Swappable config — flip on the day the option-stacking vote lands.

vContracts Pipeline

1
cap_calendar.from_team() stub
Build forward ledger from players_contract: hard committed $ per future year, existing balloon weights, expiring deals.
2
extension_filter (gatekeeper) stub
Survivors = expiring within horizon AND clear VOS threshold AND no internal replacement ETA before extension would end. "No extension" is a first-class output.
3
AAVContext.suggest_contract() reuse
Per candidate → (years, aav, total). Decline-cutoff heuristic (longest L where year-L value ≥ floor_ratio × year-1). To refactor out of free_agent_market.py into shared market_valuation.py.
4
builder.build_min_guaranteed() stub
Per candidate → ContractOffer. Distribute target AAV using the legal structure that commits least guaranteed money. Applies age-driven shape lock.
5
validator.validate() done
Reject any offer that violates the active rules profile. Catches stacked options under the post-vote rule, length caps, shape caps, buyout %.
6
cap_calendar.feasible() stub
For each candidate offer at each candidate signing year: does the balloon land on a year already at the ceiling? Reject; retry with different signing year or shape.
7
planner (joint optimization) stub
Search over (signing_year × length × shape) across all candidates simultaneously. Maximize headline TCV under: validator clean, calendar feasible, no future year exceeds balloon ceiling.

Outputs

multi-year extension schedule primary
Per candidate: signing_year, length, year-by-year structure, options, buyouts, incentives. Or "no extension recommended" with the reason (depth chart gatekeeper, VOS below threshold, no feasible balloon slot).
updated cap calendar primary
Forward ledger reflecting the proposed schedule: hard committed $, balloon weight, probable-option $, free space, decline space — per future season.
headline TCV vs expected cost per offer
What the agent sees (every dollar at face value) vs. probability-weighted true cost (incentives × P, options × team-side P). Gap = the exploitation surface.
contract-state overlay → depth_chart feedback
Guaranteed years remaining, option status, balloon-year flag, forced-roster flag — fed back into depth_chart so future roster construction sees the contract reality.
validator report diagnostic
Per offer: rule violations and their severity. Empty list = legal. Surfaces both the active rules profile's hard constraints and any soft warnings.
Reading the diagram. Left column is data flowing in (raw sources + the rules config). Middle column is the seven-step pipeline, in execution order. Right column is what the user sees back. Steps 1, 2, 4, 6, 7 are stubs awaiting implementation; step 3 reuses logic from free_agent_market.AAVContext (to be refactored into a shared module); step 5 is implemented and passing the smoke test. The depth-chart and vos_v2 outputs are inputs here because they're produced by independent tools the user already runs — vContracts consumes their CSVs, it doesn't re-derive them.
